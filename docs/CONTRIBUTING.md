# 贡献指南

感谢你对Python魔法学院项目的关注！我们欢迎各种形式的贡献。

## 如何贡献

### 报告问题

如果你发现了bug或有功能建议：

1. 在Issues中搜索，确保问题未被报告
2. 创建新Issue，清楚地描述问题或建议
3. 包含相关的错误信息、截图等

### 提交代码

1. Fork本仓库
2. 创建你的特性分支 (\`git checkout -b feature/AmazingFeature\`)
3. 提交你的修改 (\`git commit -m 'Add some AmazingFeature'\`)
4. 推送到分支 (\`git push origin feature/AmazingFeature\`)
5. 开启Pull Request

### 代码规范

- 使用TypeScript
- 遵循项目的代码风格
- 添加适当的注释
- 确保代码能够正常运行

### 提交信息规范

使用清晰的提交信息：

- \`feat: 添加新功能\`
- \`fix: 修复bug\`
- \`docs: 更新文档\`
- \`style: 代码格式调整\`
- \`refactor: 代码重构\`
- \`test: 添加测试\`
- \`chore: 构建/工具变动\`

## 项目改进方向

我们特别欢迎以下方面的贡献：

### 高优先级

1. **真实的Python代码执行**
   - 实现安全的代码执行环境
   - Docker容器集成
   - 云函数方案

2. **测试用例自动验证**
   - 为每个练习题添加完整的测试用例
   - 实现自动测试和评分系统

3. **成就系统**
   - 设计有趣的成就徽章
   - 实现成就解锁逻辑

4. **学习统计**
   - 添加学习时长统计
   - 代码提交历史
   - 进度可视化图表

### 中等优先级

5. **社区功能**
   - 代码分享
   - 讨论区
   - 学习笔记

6. **内容增强**
   - 添加更多练习题
   - 补充知识点
   - 视频教程链接

7. **UI/UX改进**
   - 移动端优化
   - 暗色模式切换
   - 无障碍访问

### 低优先级

8. **国际化**
   - 英文版本
   - 其他语言支持

9. **游戏化元素**
   - 排行榜
   - 学习小组
   - 挑战赛

## 添加新关卡

如果你想添加新的学习内容：

1. 编辑 \`scripts/init-levels.ts\`
2. 按照现有格式添加关卡数据
3. 包含：标题、描述、学习内容、练习题
4. 确保内容质量和准确性
5. 测试新关卡

## 改进现有内容

发现内容错误或可以改进？

1. 定位到相关文件
2. 修正错误或改进内容
3. 说明修改原因
4. 提交PR
