## ⚠️ 重要安全提醒

### 立即撤销 GitHub Token！

你刚才分享的token已经在对话中暴露，**必须立即撤销**！

#### 撤销步骤：

1. **访问 GitHub Settings**
   ```
   https://github.com/settings/tokens
   ```

2. **找到你的token**
   - 名称可能是 "token" 或其他
   - 查看最近创建的

3. **点击 "Delete" 或 "Revoke"**

4. **确认撤销**

#### 为什么要撤销？

- ❌ Token 已在对话中暴露
- ❌ 可能被他人看到和使用
- ❌ 有安全风险

✅ **撤销后不影响已推送的代码**
✅ **GitHub会继续保存你的代码**

## 🚀 下一步：部署到 Vercel

### 步骤1: 访问 Vercel

```
https://vercel.com
```

点击 **"Sign Up"** 或 **"Log In"**

### 步骤2: 使用 GitHub 登录

```
选择 "Continue with GitHub"
```

授权 Vercel 访问你的 GitHub 账户

### 步骤3: 导入项目

```
1. 点击 "Add New..." → "Project"
2. 在列表中找到 "pysty"
3. 点击 "Import"
```

### 步骤4: 配置环境变量

**必须配置**:

```
Name: DATABASE_URL
Value: postgresql://...（你的Neon数据库连接）
```

**可选配置**（如果使用认证）:

```
Name: NEXT_PUBLIC_STACK_PROJECT_ID
Value: your-stack-project-id

Name: NEXT_PUBLIC_STACK_PUBLISHABLE_CLIENT_KEY
Value: your-stack-client-key

Name: STACK_SECRET_SERVER_KEY
Value: your-stack-secret-key
```

### 步骤5: 部署

```
1. 确认配置无误
2. 点击 "Deploy"
3. 等待 2-3 分钟
4. 完成！🎉
```

## 📋 环境变量获取

### DATABASE_URL

1. 访问 https://console.neon.tech
2. 选择项目 "Pysty"
3. 点击 "Connection Details"
4. 复制 "Connection string"

**格式**:
```
postgresql://username:password@ep-xxx.us-east-1.aws.neon.tech/neondb
```

### Stack Auth（如果使用）

1. 访问 Stack Auth 控制台
2. 复制项目配置
3. 或从本地 `.env.local` 文件复制

## 🆘 遇到问题？

### 常见问题

1. **Vercel 构建失败**
   - 检查环境变量是否正确
   - 查看构建日志
   - 本地运行 `npm run build` 测试

2. **数据库连接失败**
   - 确认 DATABASE_URL 正确
   - 检查 Neon 数据库是否运行
   - 验证网络连接

3. **页面404错误**
   - 等待DNS传播（2-5分钟）
   - 重新部署
   - 检查路由配置
